# PulseDeck Freelab 发布和更新

你的更新仓库：

```text
https://github.com/holderethan7589915-lab/pulsedeck-freelab-updates
```

## 第一次给别人安装

1. 在装了 Node.js 的电脑上进入项目目录：

```powershell
cd D:\E\上位机
```

2. 安装依赖：

```powershell
npm install
```

3. 打安装包：

```powershell
npm run dist
```

4. 打包完成后，把 `dist` 里的安装包上传到 GitHub Releases：

```text
PulseDeck Freelab-Setup-0.1.0.exe
PulseDeck Freelab-Setup-0.1.0.exe.blockmap
latest.yml
```

别人第一次只需要下载并安装：

```text
PulseDeck Freelab-Setup-0.1.0.exe
```

## 在 GitHub 上怎么上传

1. 打开：

```text
https://github.com/holderethan7589915-lab/pulsedeck-freelab-updates/releases
```

2. 点击 `Create a new release`。

3. `Choose a tag` 填：

```text
v0.1.0
```

4. `Release title` 填：

```text
PulseDeck Freelab 0.1.0
```

5. 上传 `dist` 里的三个文件：

```text
PulseDeck Freelab-Setup-0.1.0.exe
PulseDeck Freelab-Setup-0.1.0.exe.blockmap
latest.yml
```

6. 点击 `Publish release`。

## 后续更新怎么发

比如要发布 `0.1.1`：

1. 修改 `package.json`：

```json
"version": "0.1.1"
```

2. 重新打包：

```powershell
npm run dist
```

3. 到 GitHub Releases 新建 release。

4. tag 填：

```text
v0.1.1
```

5. 上传新的三个文件：

```text
PulseDeck Freelab-Setup-0.1.1.exe
PulseDeck Freelab-Setup-0.1.1.exe.blockmap
latest.yml
```

6. 点击 `Publish release`。

用户打开软件后会自动检查 GitHub 上的新版本，下载完成后会提示重启安装。

## 注意

- 自动更新要用安装版，也就是 `npm run dist` 生成的 `Setup.exe`。
- `portable.exe` 便携版适合手动发给别人，但不适合自动更新。
- 每次发布新版本都必须递增 `package.json` 里的 `version`。
- 如果没有代码签名证书，Windows 可能提示“未知发布者”，这是正常现象。
